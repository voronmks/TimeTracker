⏱️ TimeTracker – Трекер времени за компьютером на 7 языках
Простой и удобный инструмент для учёта времени, проведённого за компьютером.
Позволяет запускать сессии, останавливать их, просматривать статус и историю работы.
Реализован на семи языках программирования с единым интерфейсом командной строки.
Данные хранятся в локальном файле в домашней директории.

🚀 Возможности
Старт/стоп сессий – команда start начинает учёт времени, stop завершает текущую сессию.

Статус – команда status показывает, идёт ли сессия и сколько времени уже прошло.

История – команда log выводит все завершённые сессии с датами и длительностью.

Автосохранение – данные записываются в файл ~/.timetracker.json после каждого изменения.

Цветной вывод – наглядные сообщения о действиях.

Кроссплатформенность – работает на Windows, Linux, macOS.

Расширяемость – легко добавить новые команды или изменить формат хранения.

📖 Использование
Синтаксис (единый для всех версий):

bash
<команда> <субкоманда>
Субкоманды
Команда	Описание
start	Начать новую сессию (если другая уже идёт, она будет завершена автоматически)
stop	Завершить текущую сессию и сохранить её в историю
status	Показать состояние: активна ли сессия, время начала, длительность
log	Вывести список всех завершённых сессий с датой и временем
Примеры
bash
# Начать сессию
python tracker.py start

# Посмотреть статус
python tracker.py status

# Остановить сессию
python tracker.py stop

# Показать историю
python tracker.py log

🧠 Формат хранения данных
Данные сохраняются в JSON-файле ~/.timetracker.json со следующей структурой:

json
{
  "sessions": [
    { "start": "2025-06-28T10:30:00", "end": "2025-06-28T12:15:00", "duration": 6300 },
    ...
  ],
  "current": null | { "start": "2025-06-28T14:00:00" }
}
sessions – массив завершённых сессий с временем начала, окончания и длительностью в секундах.

current – активная сессия (если есть), содержит только время начала.

✨ Дополнительные фичи
Автоматическое завершение – если при старте уже есть активная сессия, она автоматически завершается.

Удобное форматирование времени – длительность выводится в формате ЧЧ:ММ:СС.

Обработка ошибок – корректные сообщения при попытке остановить несуществующую сессию.
